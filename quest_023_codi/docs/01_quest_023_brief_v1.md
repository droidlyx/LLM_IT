# Quest 023 Brief — Annotation-Efficient Multi-Schema Biomedical RE via LLM-Distilled Synthetic Supervision

**Date**: 2026-05-19
**Status**: Draft for DS review
**Prerequisite**: Quest 022 (Silent CoT v1) delivered 2026-05-18

---

## 1. Motivation

Biomedical RE is bottlenecked by **annotation cost + schema fragmentation**, not model capability. Each benchmark (DrugProt, DDI, GDA, BioRED, BC5CDR, ChemProt, PharmGKB...) has its own hand-annotated train split, schemas don't transfer, and 30M+ PubMed abstracts sit unused.

### 1.1 What Quest 022 (v1) showed

- Silent CoT distillation on DrugProt (single schema) reached 0.696 macro F1 in-domain (positive-only eval)
- **Catastrophic full-RE collapse**: 0.696 → **0.039** when negative pairs are included. v1 never saw no_relation in training; the 113K negatives in DrugProt full-RE are all sucked into positive labels (e.g., predicts SUBSTRATE 21K times for 494 gold).
- BioRED full-RE: 0.22 → **0.105** (similar collapse pattern).

### 1.2 What BioREx (Lu et al., JBI 2023, `arXiv:2306.11189`) already does

- Unified PubMedBERT trained on **38,617 human-labeled relations across 9 datasets** (BioRED, DrugProt, DDI, BC5CDR, AIMed, HPRD50, PharmGKB, DisGeNET, EMU)
- Full-RE eval natively (dataset-specific None labels)
- Reported: DrugProt 80.8%, BioRED 79.6%, BC5CDR 69.9%, DDI 87.3% (full-RE F1)
- **Code + checkpoints public, reproducible**

BioREx is our primary comparator. Reviewers will demand it.

### 1.3 Quest 023 goal

Match or approach BioREx **with 100× fewer human labels** by using LLM-synthesized supervision on unlabeled PubMed.

---

## 2. Method (one-paragraph)

Train a single Qwen3-4B-Base + LoRA model on **~50K Sonnet-synthesized (sample, label, reasoning) tuples** spanning 6 biomedical RE schemas (~28 labels + no_relation). Synth is generated by labeling entity-pair candidates in unlabeled PubMed abstracts using Sonnet with multi-schema prompt + few-shot (~300 human-labeled exemplars from existing benchmarks' train splits). Training uses **unified-label-pool contrastive objective** (KL/CE softmax over ~30 labels per batch including no_relation), **per-step random template sampling** (prevents label-vector memorization), and **L_KL on with-reasoning Pass B** (preserves Silent CoT reasoning distillation). Inference: per-dataset template-based label embeddings + cosine argmax — schema-agnostic, no dataset identity needed.

---

## 3. Method (detailed)

### 3.1 Data pipeline (offline, before training)

**Source**: Unlabeled PubMed abstracts. Pool sources from local pubtator files (CDR, PharmGKB, BC5CDR, DisGeNET, etc.), **stripping their relation annotations**, keeping only doc + entity mentions. Exclude all PMIDs that appear in any test set.

**Per-doc Sonnet labeling**:
- Enumerate entity-pair candidates (unordered, filtered to schemas' valid type pairs)
- Build user message: doc + entity pair list
- Cached system prompt: 6 schemas (28 labels) + decision rules + 60 few-shot exemplars (~10 KB)
- Output: JSON line per pair `{pair_idx, label, reasoning}`

**No self-consistency filter** (revised from earlier draft): single Sonnet pass per doc. Rationale: student learns teacher's distribution; even uncertain teacher predictions carry signal. Pilot showed Run-1 vs Run-2 label distributions are nearly identical (max delta 4/99); the marginal quality gain (72.7% vs 64.3% gold agreement on intersection vs single) does not justify 2× cost.

**General label-audit rule** (replaces GDA-specific patch): any schema label whose definition asserts NON-association (X is NOT related to Y, no relation between X and Y, etc.) is universally merged into `no_relation`. In current 6 schemas this affects:
- GDA `Negative` → `no_relation`
- DDI `None` → `no_relation` (already)

Catch-all positive labels that assert SOME (unspecified) relation are kept (BioRED `Association`, DDI `int`, PharmGKB_Association).

**Negative coverage**: don't force balance. Let Sonnet's natural labeling (~40-50% no_relation from pilot) hold, but ensure negative diversity:
- Cross-schema: every schema contributes negatives
- Cross-entity-type: every valid type-pair has both positive and negative samples
- Cross-context: negatives come from varied doc contexts (not just same doc as positives)

**Output**: `~30K samples` (revised from 50K; reasoning: quest 022 worked with 15K single-schema, multi-schema 2× is sufficient). Each with `{pmid, doc, e1_text, e1_type, e2_text, e2_type, label, reasoning}`.

**Few-shot pool**: ~300 human-labeled exemplars (10/label × 30 labels) drawn from existing train splits:
- DrugProt train (14946 with Sonnet reasoning from quest 021)
- BioRED train, DDI train, GDA train, CDR train, PharmGKB train — sample 10/label each

### 3.2 Schema scope

| Schema | Source | Labels | Used as |
|---|---|---|---|
| DrugProt/ChemProt | DrugProt, ChemProt | 13 | train + eval |
| DDI | DDI | 4 + None | train + eval |
| GDA | GDA | 3 (incl. Negative) | train + eval |
| BioRED | BioRED, BC8 | 8 | train + eval |
| CDR | BC5CDR | 1 (CID) | train + eval |
| PharmGKB | PharmGKB | 1 (PharmGKB_Association) | train + eval |
| **AIMed (PPI)** | AIMed | 1 (interaction) | **HOLDOUT — never trained** |

Total ~30 labels + no_relation.

### 3.3 Training objective

```
For each step:
  # Sample mini-batch: mixed schemas (~B samples from cross-schema pool)
  # Build unified label pool L_unified for this batch:
  #   - All unique gold labels appearing in batch
  #   - no_relation
  #   - Optionally: ~5 random labels NOT in batch (cross-schema contrast)

  # Per-step random template sampling:
  for each label in L_unified:
    template = random.choice(label.templates)  # 1 of 5 per step
    label_emb[label] = encoder(template).last_token

  # Pass A: h_A = encoder(doc, e1, e2)
  # Pass B: h_B = encoder(doc, e1, e2, reasoning)

  # Loss 1: cosine matching + cross-schema contrast
  logits_A = cosine(h_A, label_emb_matrix)  # over L_unified
  L_CE = cross_entropy(logits_A / temperature, label_gold)

  # Loss 2: Silent CoT KL (preserved from v1)
  logits_B = cosine(h_B.stopgrad(), label_emb_matrix)
  L_KL = KL(softmax(logits_A) || softmax(logits_B / temperature))

  # Loss 3: optional L_gen (LM on reasoning, weighted small)

  L = L_CE + λ_kl · L_KL + λ_gen · L_gen
```

**Key changes vs v1**:
- L_CE denominator includes **cross-schema labels** + no_relation (not just current schema's K labels)
- **Random template sampling per step** (not fixed mean) — forces semantic learning, prevents label-vector memorization
- Same architecture, same inference path (cosine + per-dataset centering at test time)

### 3.4 Inference

Per test dataset:
- Encode 5 templates per label + no_relation → mean = label embedding
- Subtract dataset-specific frozen mean (centering)
- Encode test sample → centered embedding
- Cosine over (K + 1) labels

**Post-hoc threshold calibration** (NEW vs v1, addresses train/test negative-ratio mismatch):
- Training samples have ~50% no_relation; test sets have 85-97% no_relation
- Don't change training distribution. Instead, per-dataset on dev set, tune a threshold τ:
  - If `max(positive_logits) − logit(no_relation) < τ` → predict `no_relation`
  - Else → argmax of positive labels
- τ grid-search optimized for macro F1 on dev set (or default τ=0 if no dev available)

This separates training (learn internal pos/neg differentiation) from test-time decision (adapt to distribution shift). Threshold is interpretable and ablatable.

No dataset identity needed at inference except for the per-dataset centering vector + threshold.

## 5. Verification gates

### G1: Synth data quality (BEFORE training)
- Sonnet labeling acc on held-out test (200 samples per source schema, gold-positive subset):
  - BioRED / CDR / DrugProt-style: ≥ **0.85 macro F1**
  - DDI / PharmGKB: ≥ **0.80 macro F1**
  - GDA (after Negative→no_relation remap): ≥ **0.80**
- If any schema fails: tighten prompt for that schema, add more few-shot exemplars, or drop that schema from training
- Hard abort if 3+ schemas fail simultaneously

### G2: v3 in-domain on full-RE (CORE)
- DrugProt full-RE ≥ **0.50 macro F1** (vs v1 0.039, BioREx 0.808)
- BioRED full-RE ≥ **0.30 macro F1** (vs v1 0.105, BioREx 0.796)
- If fail: investigate training data quality / loss function

### G3: Cross-schema average
- Mean macro F1 across 6 datasets full-RE ≥ **0.45**
- Each dataset ≥ 0.20 (no collapse on any single one)

### G4: Holdout AIMed PPI (zero-shot to truly new schema)
- Macro F1 ≥ **0.40** with templates only (no training data for "interaction" label)
- This is the strict "new label" generalization test

### G5: Annotation efficiency claim
- v3 achieves ≥ **70% of BioREx** average across 4 head-to-head datasets (DrugProt, DDI, BioRED, BC5CDR)
- With 300 human labels vs BioREx's 38K → **127× annotation reduction**

| Risk | Likelihood | Mitigation |
|---|---|---|
| Sonnet labeling too noisy | Medium | G1 gate; fallback to BioREx data bootstrap |

## 9. Positioning vs BioREx (paper level)

> "BioREx (Lu et al. 2023) demonstrated that multi-schema unified training is possible for biomedical RE — using 38K human-annotated relations across 9 corpora. We ask: **can we achieve the same with 300 human annotations?** Answer: yes. By using Sonnet-as-annotator on unlabeled PubMed + reasoning-aware distillation into a 4B decoder, we reach ≥70% of BioREx full-RE performance with ~127× annotation cost reduction. Along the way we diagnose a systemic evaluation pitfall: positive-only training in prior LLM-based RE work (including our v1) catastrophically collapses (0.696 → 0.039) when tested on full RE. We release the first multi-schema synthetic biomedical RE dataset to support future work."

Differentiation:
- BioREx: full human supervision, single PubMedBERT, dataset-specific Nones
- v3: ~300 human + 50K LLM-synth, 4B decoder + Silent CoT, schema-agnostic inference, full-RE diagnosis
